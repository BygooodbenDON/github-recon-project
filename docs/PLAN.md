# GitHub 偵察機實作計畫 (Game Map)

這份地圖記錄了我們的網頁如何從「收到名字」到「顯示照片」的五個關卡。

## 🚀 資料傳遞的五個關卡

1. **第一關：寫地址（輸入文字）**
   - **動作**：使用者在格子裡打字。
   - **電腦做的事**：把字放進名為 `searchText` 的「神奇盒子」。

2. **第二關：按按鈕（吹響小喇叭）**
   - **動作**：按下「開始偵察」。
   - **電腦做的事**：聽到喇叭聲，啟動搜尋程式。

3. **第三關：叫外送（派人去 GitHub）**
   - **動作**：外送員 `fetch` 出發。
   - **電腦做的事**：帶著名字去 GitHub 大倉庫找包裹。

4. **第四關：拿包裹（換成資料）**
   - **動作**：外送員帶回資料。
   - **電腦做的事**：收到 GitHub 給的裝滿照片和資訊的大包裹。

5. **第五關：裝箱與變身（顯示畫面）**
   - **動作**：把包裹放進 `userInfo` 盒子。
   - **電腦做的事**：開啟「隱身開關」，讓照片在網頁上亮起來！
