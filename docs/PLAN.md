# GitHub 偵察機實作計畫 (Game Map)

這份地圖記錄了我們的網頁如何從「收到名字」到「顯示照片與作品」的進化過程。

## 🚀 資料傳遞的進化關卡

1. **第一關：寫地址（輸入文字）**
   - **動作**：使用者在格子裡打字。
   - **電腦做的事**：把字放進 `searchText` 盒子。

2. **第二關：按按鈕（吹響小喇叭）**
   - **動作**：按下「開始偵察」。
   - **電腦做的事**：聽到喇叭聲，啟動搜尋程式。

3. **第三關：啟動風車（讀取中）**
   - **動作**：打開 `isLoading` 開關。
   - **電腦做的事**：顯示旋轉燈，告訴使用者「外送員出發囉！」

4. **第四關：叫外送（派人去 GitHub）**
   - **動作**：外送員 `fetch` 出發，帶著「專屬地址」去拿資料。
   - **電腦做的事**：先拿個人名片，再跑一趟拿作品清單。

5. **第五關：拿包裹（換成資料）**
   - **動作**：外送員帶回兩個大包裹。
   - **電腦做的事**：把名片放進 `userInfo`，把作品放進 `userRepos`。

6. **第六關：排隊報數（顯示清單）**
   - **動作**：使用「排排站指令」 `v-for`。
   - **電腦做的事**：讓作品一個個整齊地排在畫面上。

7. **第七關：任務完成（變身成功）**
   - **動作**：關閉風車，撥開隱身開關。
   - **電腦做的事**：讓帥氣的深色名片和作品集呈現在畫面上！
